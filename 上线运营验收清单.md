# 哇噻·高黎贡预约系统 · 上线与研学运营验收清单

## A. 部署前

- [ ] GitHub 仓库已创建，默认分支为 `main`，未提交 `data.json`、密码或生产 token。
- [ ] Render/VPS 使用 Node 18+；设置 `ADMIN_TOKEN` 为随机值；启用 HTTPS。
- [ ] `/healthz` 返回 `{"ok":true}`；前台 `/` 和后台 `/管理后台` 均可打开。
- [ ] 已建立 `data.json` 定期备份和恢复演练；明确数据文件不适合规模化生产。

## B. 发布验收

- [ ] 新用户注册后，服务端 `data.json` 中只有 `pwdHash`，没有 `pwd`。
- [ ] 错误密码、错误后台 token、已取消凭证均被拒绝。
- [ ] 每个活动场次可查看容量；重复预约、超容量预约被拒绝。
- [ ] 预约详情包含 `participants` 和 `credentialPayload`；用户可取消待核销预约。
- [ ] 后台可核销有效凭证，重复核销被拒；SSE 实时更新正常。

## C. 7/18–19 研学放票前置门

- [ ] 爽约占号压测 < 8%。
- [ ] 双码核验无超时、无重复核销；弱信号时启用纸质名单和手工台账。
- [ ] 候补补位延迟 < 5 分钟；取消释放名额可回填。
- [ ] 师资、助教、安全员、核验岗、引流岗已按场次排班。
- [ ] 雨备馆、专属动线、低龄防走失、动物 enclosure 隔离、医疗绿通已联合点检签字。
- [ ] 对外统一使用“购票赠课”，不写成“免票进园”或另收研学费。

## D. 外部依赖

- [ ] GitHub push：需要仓库 URL 和具备 `repo` 权限的 PAT 或已登录平台。
- [ ] Render/VPS：需要用户账号权限；本地代码无法代替控制台创建服务。
- [ ] 微信订阅消息、openid、短信、支付：需要对应主体资质与第三方账号，本版本未接入。
