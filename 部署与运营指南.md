# 哇噻·高黎贡·自然纪 · 预约小程序 — 部署与设置指南

> 这是一个**纯 Node、零依赖**的小程序：一个 `node 服务器.js` 同时托管前台（`用户前台.html`）、后台（`管理后台.html`）和所有 API / SSE 实时推送。
> **关键认知**：它不是静态网页，必须有一个**正在运行的 Node 服务**才能用（尤其是后台的数据总览 / 核销 / 实时）。

---

## 一、为什么微信打不开 / 后台不便？（先读这段）

| 现象 | 根因 | 解决 |
|------|------|------|
| 微信里打开 `localhost:8090` 没反应 | `localhost` 指向**手机自己**，不是你的电脑。手机访问不到你电脑上的服务 | 用下面「方案 A/B」把服务放到**公网**；或先用「LAN 本机自测」 |
| 之前的云链接能打开但后台（数据/核销/实时）用不了 | 之前的云部署是**纯静态**（只传了 html，没跑 Node），所以没有后端 | 必须把 **整个 Node 应用** 部署到有 Node 运行时的主机（见方案 A/B） |
| 云链接"打不开" | 静态工作空间会休眠/过期；且本质无后端 | 重新部署静态仅作"浏览演示"，**完整功能仍需方案 A/B** |

**结论**：要让微信 anywhere 打开且后台可用，唯一缺的"设置"就是——**把 `服务器.js` 跑在一个公网可达的 Node 主机上**，然后用它给的 `https` 地址。

---

## 二、方案 A：本机 + 同 Wi-Fi 快速自测（不花钱，立刻能用）

1. 电脑运行：`node 服务器.js`（默认 `http://localhost:8090`）。
2. 查电脑局域网 IP：Windows `ipconfig` 看「IPv4 地址」（如 `192.168.1.20`）。
3. 手机连**同一个 Wi-Fi**，微信里打开 `http://192.168.1.20:8090`。
4. 后台：`http://192.168.1.20:8090/管理后台`，token = `admin888`。

> 注意：部分手机微信对纯 `http` 局域网地址会拦截，可改用手机浏览器测试；正式对外请用方案 B（https）。

---

## 三、方案 B：部署到公网 Node 主机（推荐，微信 anywhere 可用）

### 方式 1 · Render（最快，免费额度够用）

**A. 一键蓝图（推荐）**：仓库根已含 `render.yaml`。
1. 把本目录推到 GitHub 仓库（见第七节）。
2. 打开 https://render.com → New → **Blueprint** → 关联仓库 → **Apply**。
3. Render 读取 `render.yaml`：自动建 Web Service、`node 服务器.js` 启动、`/healthz` 健康检查、注入 `PORT`。
4. 自动分配 `https://xxxx.onrender.com`，微信打开即可，后台加 `/管理后台`。

**B. 手动 Web Service**（不用 yaml 也行）：
1. 打开 https://render.com → New → Web Service → 关联仓库。
2. 设置：**Build Command** 留空（零依赖）；**Start Command**：`node 服务器.js`；**Instance Type**：Free。
3. 自动分配 https 地址。

### 方式 2 · 自有服务器 / VPS
```bash
# 1) 上传本目录到服务器
scp -r . user@your-server:/opt/booking/

# 2) 安装进程守护（任选其一）
npm i -g pm2
pm2 start 服务器.js --name booking
pm2 save && pm2 startup

# 3) 开放端口（如 8090），并用 Nginx 反代 + HTTPS（Let's Encrypt / Cloudflare）
```
Nginx 示例（反代 + https）：
```nginx
server {
  listen 443 ssl;
  server_name book.your-domain.com;
  location / {
    proxy_pass http://127.0.0.1:8090;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  }
}
```

## 四、微信相关设置（重要）

- 本应用是**普通 H5 网页**，在微信内置浏览器打开**只需一个公网 https 地址**，**不需要**微信 JS-SDK 鉴权、不需要公众号后台配置"JS接口安全域名"，**除非**你后续要接微信支付 / 分享卡片 / 定位等原生能力。
- 若未来要接微信支付：需在公众号/商户平台配置「支付授权目录 / 回调域名」，那是另一套配置，本版未涉及。
- 二维码：把公网地址生成二维码，印在海报/窗口即可扫码进园预约。

## 五、运营必备参数

| 项 | 值 |
|----|----|
| 前台地址 | `/`、`/前台` 或 `/index.html` |
| 后台地址 | `/管理后台`、`/admin` 或 `/admin.html` |
| 后台口令（token） | 由 `ADMIN_TOKEN` 环境变量提供；Render 首次部署自动生成 |
| 端口 | `PORT` 环境变量，默认 `8090` |
| 数据文件 | `data.json`（JSON 文件持久化，重启不丢） |
| 健康检查 | `/healthz` → `{"ok":true}` |
| 实时流 | `/api/stream?token=<ADMIN_TOKEN>`（SSE，后台实时刷新） |

用户注册/登录会获得 7 天 session；个人数据、预约、取消、投票和投稿接口均需携带 `Authorization: Bearer <session>`。预约成功和取消会生成站内通知，在前台「我的」页面显示。

## 六、常用命令

```bash
node 服务器.js            # 启动（生产的就是它）
node 研学计划种子.js         # 把「研学活动计划」写入系统（可重复执行，幂等）
# 重置数据：删除 data.json 后重启，将回到内置默认活动
node 生产验证.js         # MVP 安全/容量/取消/家庭成员 smoke test
# PowerShell：$env:TEST_BASE='http://127.0.0.1:8090'; node 回归验证.js; node 新功能验证.js
```

> 安全提示：生产环境必须设置强随机 `ADMIN_TOKEN` 并使用 HTTPS；用户密码以 scrypt 哈希保存，`data.json` 仍含手机号/脱敏身份信息和预约数据，必须限制文件权限并定期备份。当前 JSON 数据层适合小范围 MVP，不适合高并发生产规模。

## 七、推送到 GitHub（本目录即仓库根，已就绪）

本目录已是「可直接推 GitHub」的结构：

| 文件 | 作用 |
|------|------|
| `package.json` | 声明 `start=node 服务器.js`、engines node>=18 |
| `render.yaml` | Render Blueprint，控制台一键 Apply |
| `Procfile` | `web: node 服务器.js`（兼容 Heroku 等） |
| `.gitignore` | 已忽略 `data.json`（运行时隐私数据，不入库） |
| `服务器.js` 内置 `defaultDB()` | 首次启动无 `data.json` 时自动创建，直接带**完整研学计划（四场 + 三草稿）** |

推送步骤（在 booking-miniprogram 目录内执行）：
```bash
git init
git add .
git commit -m "哇噻·高黎贡·自然纪 预约小程序 v1"
git remote add origin <你的仓库地址>
git push -u origin main
```
推完即可在 Render 用 Blueprint 一键部署（见方式 1·A）。

> 提示：`data.json` 已被忽略，团队成员 clone 后 `node 服务器.js` 会得到一份干净的默认数据（含研学计划），互不影响；若要导入你当前的运营数据，把现有 `data.json` 单独拷贝到部署机即可（**不要**提交进 git）。
